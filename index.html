개인 프로젝트

포트폴리오
생성 서비스

취업준비생의 프로젝트 경험을 AI 기반으로 포트폴리오 카드로 자동 변환

2026.02.23 ~ 2026.03.13

기획 · 풀스택 · 서비스 운영

Tech Stack

Next.js NestJS TypeScript LLM API Prompt Engineering React REST API Git

프로젝트 내용

  • 기획 배경: 취업준비생이 프로젝트 경험을 체계적으로 정리하는 데 어려움을 겪는다는 문제 인식에서 출발
  • 사용자가 최소한의 입력만으로 정돈된 포트폴리오 카드를 생성할 수 있도록 AI 연동 서비스를 1인 개발
  • 프로젝트 개요, 기술 선택 이유, 문제 상황, 해결 방식, 성과 등 문맥 정보를 구조화해 LLM에 전달하는 파이프라인 설계
  • 결과 형식과 필수 요소가 빠지지 않도록 프롬프트 엔지니어링 고도화
  • 기획 → 개발 → 테스트 → 베타검증 → 출시 전 과정 1인 수행

트러블슈팅

사용자가 적게 입력할수록 AI 결과물의 일관성이 낮아지고 핵심 요소가 누락되는 문제 발생
컨텍스트를 6개 항목(개요·기술선택·문제·해결·성과·결과형식)으로 구조화하고, 프롬프트에 필수 출력 요소를 명시적으로 지정하여 결과 품질 표준화
초기 AI 응답에서 형식이 제각각이어서 카드 UI 렌더링에 오류 발생
응답 스키마를 JSON 형식으로 강제하고, 파싱 실패 시 재요청 로직 추가 및 fallback UI 처리

개선 및 성과

30+ 반복 테스트 횟수
5명 베타테스터 검증
50명 직업훈련 교육생 출시
4.1점 만족도 (목표 3.5점)
  • 레포지토리 기준 30회 이상 반복 테스트 후 베타테스터 5명 대상 불편 요소·결과 품질 점검, 피드백 반영
  • 직업훈련 교육생 50명에게 정식 출시, 설문 목표(3.5점) 대비 평균 4.1점 달성
  • 기능 구현을 넘어 사용자 평가로 품질을 증명하는 개발 사이클 직접 완수

Tech Stack

프로젝트 내용

  • 게시글·커뮤니티 기능에 더해 실시간 채팅이 결합된 개발자 소통 플랫폼
  • 담당 파트: 채팅 기능 전반 구현 + 소켓 서버 개발
  • 실시간 메시지 송수신 구조 설계: 클라이언트-서버 간 이벤트 흐름 연결
  • 다중 사용자 동시 접속 환경에서도 안정적으로 메시지가 전달되는 처리 구조 구현
  • 채팅방 입장·퇴장·메시지 브로드캐스트 이벤트 설계 및 서버 사이드 에러 핸들링

트러블슈팅

다수 사용자가 동시에 채팅방에 접속할 때 메시지 순서가 뒤바뀌거나 일부 클라이언트에 미전달되는 문제
소켓 룸(room) 단위로 메시지 브로드캐스트를 분리하고, 서버 이벤트 큐 처리 순서를 보장하는 구조로 재설계
클라이언트 재연결 시 이전 채팅 히스토리가 유실되는 UX 문제
연결 시점에 최근 메시지 n개를 서버에서 즉시 전송하는 초기화 이벤트 추가, 사용자 경험 개선

개선 및 성과

  • 단순 화면 구현을 넘어 실시간 통신 구조와 서버 처리 흐름에 대한 이해를 직접 확장
  • 클라이언트와 서버 양쪽을 함께 설계하며 풀스택 관점의 아키텍처 사고력 향상
  • 사용자 간 상호작용이 핵심인 서비스의 실시간 기능을 실서비스 수준으로 완성

잔여 기간·보유 기술·역할별 구현 범위를 재점검해 드립 커피 레시피 생성 서비스로 주제 전환. MVP 기준 재정의 후 일정 안에 완성

공공 API 응답 속도 지연 및 간헐적 오류로 인한 UI 미응답 상태
로딩 상태 UI 및 에러 fallback 처리를 추가해 사용자에게 시각적 피드백 제공
